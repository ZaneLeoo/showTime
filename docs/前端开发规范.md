## 二、API 层规范

### 2.1 接口封装

`request.ts` 暴露四个方法，所有 API 调用必须通过它们：

```typescript
import { get, post, put, del } from '@/api/request'

// GET 请求
get<T>(url, params?) → Promise<T>

// POST 请求
post<T>(url, data?, config?) → Promise<T>

// PUT 请求
put<T>(url, data?) → Promise<T>

// DELETE 请求
del<T>(url) → Promise<T>
```

`T` 是 `Result.data` 的类型，方法内部已解包，直接返回 data。

### 2.2 API 文件写法

```typescript
// api/event.ts
import { get } from './request'
import type { EventCard, PageData } from '@/types/common'

export function getEventList(params: {
  categoryId?: number
  city?: string
  keyword?: string
  page?: number
  pageSize?: number
}): Promise<PageData<EventCard>> {
  return get<PageData<EventCard>>('/event/list', params)
}
```

规则：
- 一个 API 文件对应一个后端模块（`user.ts` / `event.ts` / `order.ts`）
- 函数名动词开头：`get` / `create` / `update` / `delete`
- 参数和返回值必须有类型

### 2.3 组件中调用

```vue
<script setup lang="ts">
import { ref, onMounted } from 'vue'
import { getEventList } from '@/api/event'
import type { EventCard } from '@/types/common'

const list = ref<EventCard[]>([])
const loading = ref(false)

onMounted(async () => {
  loading.value = true
  try {
    const data = await getEventList({ page: 1, pageSize: 20 })
    list.value = data.records
  } catch (e) {
    // 错误已在拦截器统一处理，这里只需处理 UI 反馈
    console.error(e)
  } finally {
    loading.value = false
  }
})
</script>
```

### 2.4 拦截器行为

- **请求拦截**：自动从 `localStorage` 取 token 加到 `Authorization` 头
- **响应拦截**：
  - `code === 200` → 正常返回
  - `code === 4001` → 自动清 token 跳转登录
  - 其他错误码 → `Promise.reject`
- **不需要手动判断 code**，组件侧 `catch` 即可

## 三、类型定义规范

### 3.1 类型来源

```typescript
// types/common.ts — 所有 TS 类型集中管理

// 按模块分段，用注释分隔
// ==================== 用户相关 ====================
export interface LoginReq { ... }
export interface LoginRes { ... }

// ==================== 演出相关 ====================
export interface EventCard { ... }
```

### 3.2 类型对应

| 后端 Java 类 | 前端 TS 类型 |
|-------------|-------------|
| `Result<T>` | `Result<T>` (common.ts) |
| `PageResult<T>` | `PageData<T>` (common.ts) |
| `EventCardVO` | `EventCard` (common.ts) |
| `EventPageDTO` | 函数参数内联 interface |

- 前端不写 `VO` / `DTO` 后缀
- 简单参数直接用内联 interface，不单独命名

## 四、路由规范

### 4.1 路由定义

```typescript
{
  path: '/event/:id',
  name: 'event-detail',       // kebab-case
  component: () => import('@/views/EventDetail.vue'),  // 懒加载
  meta: { requiresAuth: true },  // 需要登录
}
```

### 4.2 路由命名

| 项目 | 规则 | 示例 |
|------|------|------|
| path | 小写 + 连字符 | `/event/:id/seat` |
| name | kebab-case | `seat-select` |
| 文件 | PascalCase | `SeatSelect.vue` |

## 五、状态管理（Pinia）

```typescript
// stores/auth.ts
export const useAuthStore = defineStore('auth', () => {
  // state 用 ref
  const token = ref<string>('')

  // getter 用 computed（如需）或普通函数
  const isLoggedIn = () => !!token.value

  // action 用普通函数
  async function login(data: LoginReq) { ... }

  return { token, isLoggedIn, login }
})
```

规则：
- 使用 Composition API 风格（`defineStore('name', () => {})`)
- Store 名：`useXxxStore`
- 状态字段尽量精简，不存计算值

## 六、样式规范

### 6.1 优先使用 Tailwind

```vue
<!-- ✅ 好：Tailwind 原子类 -->
<div class="flex items-center gap-4 p-6 bg-gray-900 rounded-xl">

<!-- ❌ 避免：裸写自定义 CSS（除非 Tailwind 做不到） -->
<div class="my-custom-layout">
```

### 6.2 必须自定义 CSS 时

在 `<style scoped>` 中写，或放到 `styles/` 下作为全局工具类。

### 6.3 设计约定

- 暗色主题为主（`bg-gray-950 text-white`）
- 品牌色：紫到粉渐变（`brand` / `accent` 色表在 tailwind.config.js）
- 圆角卡片风格
- 过渡动效用 Tailwind `transition-*`

## 七、组件规范

### 7.1 分类

| 类型 | 位置 | 说明 |
|------|------|------|
| 页面 | `views/` | 一个路由对应一个页面 |
| 公共组件 | `components/` | 跨页面复用 |
| 页面私有组件 | `views/` 同级目录 | 只被一个页面使用的，可以不放 components |

### 7.2 组件编写

```vue
<template>
  <!-- 模板 -->
</template>

<script setup lang="ts">
// 始终使用 <script setup>
// Props 用 defineProps
// Emits 用 defineEmits
</script>

<style scoped>
/* 仅在 Tailwind 无法满足时写 */
</style>
```

### 7.3 命名

- 组件文件：`PascalCase`（`EventCard.vue`, `SeatMap.vue`）
- 组件内变量/函数：`camelCase`
- 常量：`UPPER_SNAKE_CASE`

## 八、关键约定

1. **不要手动处理 token**：拦截器已自动处理，除非特殊场景
2. **不要硬编码 baseURL**：用 `/api` 代理，vite.config.ts 已配置
3. **类型必须写**：所有 API 函数声明返回值和参数类型
4. **懒加载**：路由组件全部用 `() => import()` 动态导入
5. **一个 API 文件一个业务模块**：不要把用户和订单 API 混在一个文件
