## 1. 知识清单

### 视图 VIEW

```sql
-- 创建
CREATE VIEW view_name AS
SELECT ... FROM ... JOIN ... WHERE ...;

-- 查询（和查表一样）
SELECT * FROM view_name WHERE ...;

-- 删除
DROP VIEW view_name;

-- 可更新视图（满足条件时可以通过视图 UPDATE 基表）
CREATE VIEW v_available_seats AS
SELECT * FROM seats WHERE status = 'available'
WITH CHECK OPTION;  -- 防止 UPDATE 后行从视图中消失
```

**视图的用途**：封装复杂查询、做权限隔离（给报表用户只读视图不给表权限）

### 存储过程 PROCEDURE

```sql
-- 创建（无参数）
CREATE PROCEDURE proc_name()
BEGIN
    SQL语句;
END;

-- 创建（带参数）
CREATE PROCEDURE proc_name(
    IN param1 INT,        -- 输入参数
    OUT param2 VARCHAR,   -- 输出参数
    INOUT param3 DOUBLE   -- 输入输出参数
)
BEGIN
    -- 变量声明
    DECLARE var_name TYPE DEFAULT value;
    -- 游标
    DECLARE cur CURSOR FOR SELECT ...;
    -- 条件处理
    DECLARE EXIT HANDLER FOR SQLEXCEPTION ...;
    -- 逻辑
    ...
END;

-- 调用
CALL proc_name(1, @out_val);
SELECT @out_val;

-- 删除
DROP PROCEDURE proc_name;
```

### 自定义函数 FUNCTION

```sql
CREATE FUNCTION func_name(param TYPE) RETURNS return_type
DETERMINISTIC          -- 确定性函数（相同输入永远相同输出）
READS SQL DATA         -- 或 NO SQL / MODIFIES SQL DATA
BEGIN
    DECLARE result TYPE;
    -- 逻辑
    RETURN result;
END;

-- 调用
SELECT func_name(100) FROM DUAL;
-- 或在查询中
SELECT name, func_name(price) FROM products;
```

### 触发器 TRIGGER

```sql
CREATE TRIGGER trigger_name
BEFORE|AFTER INSERT|UPDATE|DELETE ON table_name
FOR EACH ROW
BEGIN
    -- NEW.column：新行的列值（INSERT/UPDATE）
    -- OLD.column：旧行的列值（UPDATE/DELETE）
    INSERT INTO audit_log (action, old_val, new_val, op_time)
    VALUES ('UPDATE', OLD.status, NEW.status, NOW());
END;

-- 查看触发器
SHOW TRIGGERS;
-- 删除
DROP TRIGGER trigger_name;
```

## 2. ShowTime 业务场景

### 场景 1：报表视图 — 票房日报

```sql
-- 经常要查每日销售额，每次写 JOIN 很麻烦
-- → 封装成视图，后端直接 SELECT * FROM view_name WHERE date=?

CREATE VIEW v_daily_sales AS
SELECT 
    DATE(p.pay_time) AS sale_date,
    COUNT(DISTINCT o.id) AS order_count,
    COUNT(ot.id) AS ticket_count,
    SUM(ot.price) AS total_amount
FROM payments p
JOIN orders o ON p.order_id = o.id
JOIN order_tickets ot ON o.id = ot.order_id
WHERE p.status = 'paid'
GROUP BY DATE(p.pay_time);
```

### 场景 3：触发器 — 审计日志

```sql
-- 需求：任何订单状态变更都要记录到 audit_log 表
-- 记录：谁改的、什么时候改的、从什么状态改成什么状态

CREATE TRIGGER trg_orders_audit
AFTER UPDATE ON orders
FOR EACH ROW
BEGIN
    IF OLD.status != NEW.status THEN
        INSERT INTO audit_log (table_name, record_id, field_name, 
                               old_value, new_value, changed_at)
        VALUES ('orders', NEW.id, 'status', 
                OLD.status, NEW.status, NOW());
    END IF;
END;
```

### 场景 4：触发器 — 支付后自动更新订单状态

```sql
-- 需求：payments 表 INSERT 一条已支付记录后
-- 自动把 orders.status 更新为 'paid'

CREATE TRIGGER trg_payment_after_insert
AFTER INSERT ON payments
FOR EACH ROW
BEGIN
    IF NEW.status = 'paid' THEN
        UPDATE orders SET status = 'paid', paid_at = NOW()
        WHERE id = NEW.order_id;
    END IF;
END;
```

### 场景 5：自定义函数 — 计算折扣价

```sql
-- 需求：不同座位区域不同折扣策略
-- VIP 不打折，A 区 9 折，B 区 8.5 折，C 区 8 折
-- 早鸟票（开演前 30 天）额外再打 9.5 折

CREATE FUNCTION calc_ticket_price(
    original_price DECIMAL(10,2),
    category_name VARCHAR(20),
    event_date DATE
) RETURNS DECIMAL(10,2)
DETERMINISTIC
BEGIN
    DECLARE final_price DECIMAL(10,2);
    -- 区域折扣
    SET final_price = CASE category_name
        WHEN 'VIP' THEN original_price
        WHEN 'A' THEN original_price * 0.9
        WHEN 'B' THEN original_price * 0.85
        WHEN 'C' THEN original_price * 0.8
        ELSE original_price
    END;
    -- 早鸟折扣
    IF DATEDIFF(event_date, CURDATE()) >= 30 THEN
        SET final_price = final_price * 0.95;
    END IF;
    RETURN ROUND(final_price, 2);
END;
```

## 4. 关键提醒

- **视图只是查询的别名**，不存数据（除非物化视图），每次查视图都是实时执行 SQL
- **存储过程中不要放业务逻辑**：只适合数据密集型的批量操作。判断用户权限、格式化数据这些应该在应用层做
- **触发器是隐式执行的**：调试困难，出问题不容易发现。一般只用于审计日志、数据校验这种「必须和 DML 绑在一起」的场景
- **MySQL 函数 vs 存储过程**：函数有返回值，可以在 SELECT 里调用；存储过程用 OUT 参数返回，用 CALL 执行
- **生产上很多团队禁用触发器和存储过程**：逻辑散落在数据库里不好维护。但学习阶段要会写，理解它的作用
