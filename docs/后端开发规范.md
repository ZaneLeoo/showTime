# ShowTime 后端开发规范

## 一、包结构

```
com.showtime/
├── ShowTimeApplication.java      # 启动类
├── common/                       # 公共类
│   ├── Result.java               # 统一返回
│   ├── ResultCode.java           # 响应码枚举
│   ├── PageResult.java           # 分页返回
│   ├── BizException.java         # 业务异常
│   └── GlobalExceptionHandler.java
├── config/                       # 配置类
├── controller/                   # 接口层
├── service/                      # 业务接口
│   └── impl/                     # 业务实现
├── mapper/                       # MyBatis-Plus Mapper
├── entity/                       # 数据库实体
├── dto/                          # 请求体 DTO
└── vo/                           # 响应体 VO
```

## 二、统一返回

### 2.1 返回结构

所有接口统一返回 `Result<T>`：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": { ... }
}
```

### 2.2 Controller 写法

```java
// 返回单个对象
@GetMapping("/{id}")
public Result<EventVO> getDetail(@PathVariable Long id) {
    EventVO vo = eventService.getDetail(id);
    return Result.ok(vo);
}

// 返回分页
@GetMapping("/list")
public Result<PageResult<EventCardVO>> list(EventPageDTO dto) {
    IPage<EventCardVO> page = eventService.pageQuery(dto);
    return Result.ok(PageResult.from(page));
}

// 返回无数据（如删除/修改）
@PutMapping("/{id}")
public Result<Void> update(@PathVariable Long id, @Valid @RequestBody EventSaveDTO dto) {
    eventService.update(id, dto);
    return Result.ok();
}
```

### 2.3 不要这样写

```java
// ❌ 不要自己包装 Map
@GetMapping
public Map<String, Object> list() { ... }

// ❌ 不要在 Controller 里捕获异常
@GetMapping("/{id}")
public Result<?> get(@PathVariable Long id) {
    try {
        ...
    } catch (Exception e) {
        return Result.fail(500, e.getMessage());
    }
}

// ❌ 不要直接用 entity 返回
@GetMapping("/{id}")
public Result<Events> get(@PathVariable Long id) { ... }
```

## 三、响应码

| 范围 | 含义 |
|------|------|
| 200 | 成功 |
| 4000-4099 | 客户端错误（参数、权限、登录） |
| 4100-4199 | 业务错误（座位、订单） |
| 5000-5099 | 服务端错误 |

使用 `ResultCode` 枚举，不要硬编码数字。

### 新增错误码

```java
// 在 ResultCode.java 中按段添加
SEAT_SOLD(4020, "该座位已售出"),
ORDER_EXPIRED(4032, "订单已过期，请重新下单"),
```

## 四、异常处理

### 4.1 业务异常直接抛

```java
// Service 层遇到业务规则不满足
if (seat.getStatus() != SeatStatus.AVAILABLE) {
    throw new BizException(ResultCode.SEAT_UNAVAILABLE);
}
```

`GlobalExceptionHandler` 会自动捕获，转成 `Result.fail()` 返回。

### 4.2 参数校验用 Validation

```java
// DTO 中
@Data
public class RegisterDTO {
    @NotBlank(message = "手机号不能为空")
    @Pattern(regexp = "^1[3-9]\\d{9}$", message = "手机号格式不正确")
    private String phone;

    @NotBlank(message = "密码不能为空")
    @Size(min = 6, max = 20, message = "密码长度6-20位")
    private String password;
}

// Controller 中
@PostMapping("/register")
public Result<Void> register(@Valid @RequestBody RegisterDTO dto) {
    userService.register(dto);
    return Result.ok();
}
```

校验失败时 `GlobalExceptionHandler` 自动返回 4000 错误。

## 五、分层职责

### Controller
- **只做**：接收参数、调用 Service、返回 Result
- **不做**：业务判断、直接调 Mapper、拼 SQL

### Service
- **做**：所有业务逻辑、事务控制
- **事务**：写操作加 `@Transactional(rollbackFor = Exception.class)`

### Mapper
- 继承 `BaseMapper<Entity>`
- 简单 CRUD 直接用 MyBatis-Plus 方法
- 复杂查询写 XML（放在 `resources/mapper/`）

## 六、实体类规范

### 6.1 Entity（对应数据库表）

```java
@Data
@TableName("events")
public class Event {
    @TableId(type = IdType.AUTO)
    private Long id;
    private String title;
    private Long categoryId;
    // ...
}
```

- 类名驼峰、表名下划线（MyBatis-Plus 默认映射）
- 主键用 `@TableId(type = IdType.AUTO)`
- 用 Lombok `@Data`

### 6.2 DTO vs VO

| 类型 | 方向 | 用途 |
|------|------|------|
| DTO | 前端 → 后端 | 请求参数封装 |
| VO | 后端 → 前端 | 响应数据裁剪 |
| Entity | 内存 ↔ 数据库 | 与表一一映射 |

```java
// DTO：接收请求参数
@Data
public class EventPageDTO {
    private Integer categoryId;
    private String city;
    private String keyword;
    private Integer page = 1;
    private Integer pageSize = 20;
}

// VO：裁剪返回字段（避免返回整张 entity）
@Data
public class EventCardVO {
    private Long id;
    private String title;
    private String posterUrl;
    private String venueName;
    private String city;
    private BigDecimal minPrice;
    private LocalDateTime earliestTime;
}
```

## 七、命名规范

| 类型 | 规则 | 示例 |
|------|------|------|
| Entity | 单数名词 | `Event`, `Order`, `Seat` |
| Mapper | Entity+Mapper | `EventMapper` |
| Service 接口 | Entity+Service | `EventService` |
| Service 实现 | Entity+ServiceImpl | `EventServiceImpl` |
| Controller | Entity+Controller | `EventController` |
| DTO | 功能+DTO | `EventPageDTO`, `OrderCreateDTO` |
| VO | 功能+VO | `EventCardVO`, `OrderDetailVO` |
| URL | 复数名词 | `/events`, `/orders` |

## 八、URL 设计

```
GET    /events              → 演出列表（分页+筛选）
GET    /events/{id}         → 演出详情
POST   /events              → 新增演出（管理）
PUT    /events/{id}         → 更新演出（管理）

GET    /events/{id}/sessions     → 某演出的所有场次
GET    /sessions/{id}/seats      → 某场次的所有座位
POST   /sessions/{id}/seats/lock → 锁定座位

GET    /orders              → 我的订单列表
POST   /orders              → 创建订单
GET    /orders/{id}         → 订单详情
POST   /orders/{id}/pay     → 支付订单

POST   /user/register       → 注册
POST   /user/login          → 登录
GET    /user/info           → 当前用户信息
```
